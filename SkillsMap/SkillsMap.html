前言

如何学习新技术？

  1. 快速了解：看视频，入门 blog，专题 blog。

  2. 解读代码：去搞清楚每一行代码，并笔记记录。

  3. 代码选择：github 上的 code tutrial 库最佳，零散代码亦可。

  4. 不是为了保留代码，是为了记住、理解代码。

  5. code tutorial 库可长期维护，温故知新。

  6. 系统学习：看书，看官方文档，做笔记。 在掌握技术的用法后，看一本书（不要太多的书）系统的梳理知识点。

  7. 总结笔记：可快速复习。

CTO 职责

  1. 定战略（制定技术战略）：结合业务发展需求和现状，制定技术发展战略、对系统架构发展进行规划、组织制定和实施重大技术决策和方案（人工智能、数据中台等）；
  2. 搭班子（团队建设）：定岗位明职责、制定人才发展晋升通道、按职位制定技术学习方案、招聘、培养培训
  3. 带队伍（执行力）：
  4. 整体规划、分层细化、组织人力团队、定职责、监督与指导、工作评价
  5. 三评审：需求评审、计划评审、代码评审
  6. 制定研发流程：主导制定并持续优化研发工作流程和规范，保证高效率高质量的技术产出，促进技术成果的沉淀共享，确保技术成果的有效性和持续性；
  7. 制定研发技术路线：
  8. 制定或评审技术架构和数据库设计；
  9. 优化现有技术框架；
  10. 负责部门软件构件库的搭建、第三方控件的选型；
  11. 保障业务：最大程度支撑部门产品的技术实现和稳定运行，力求技术稳定性和先进性为部门发展提供动力，容量规划预备业务的快速增长；完成各项技术工作相关数据统计分析报告，并据此对部门发展提出合理化建议；
  12. 新技术研究：研究部门战略发展相关最新产品和技术的发展方向，调研与产品相符的最新技术，并在系统中使用以提升产品体验。
